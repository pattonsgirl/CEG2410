## Dotfiles Peer Review

Your task is to review a classmates dotfiles collection.  You will provide feedback by generating Issues or commenting on existing Issues in their repository.

Review a minimum of two other repositories. You may only comment on an existing issue once, otherwise you need to review and create a new issue.

### Types of Issues (Suggestions)

1. Missing features.
    - Review [assignment requirements](https://github.com/pattonsgirl/CEG2410/blob/main/Assignments/dotfiles) and check that their collection has a minimum of the requirements.  Note anything missing in an Issue
2. Organization feedback.
    - Common organization issues can include
        - Lack of repository documentation - no README.md defining what the collection does or how it is organized
        - No folder organization.  Sure, it made path references easier in the scripts, but no one likes to look at a dump of files
3. Logic / structure feedback.
   - Everything is in one monolithic script.  As we discussed, this can make challenges if you need to setup what you can without root privileges - what if you just want to set up your user account!
   - Lack of script comments or output statements to explain to users what is happening as the script runs
   - Conditions worth checking.  Sometimes your script should ask before auto-doing a task.  This can include yes/no questions of doing an action, or existence conditions like specific directories

## Submission

In the dropbox on Pilot, links to the Issues you generated in the repositories they were added to.
